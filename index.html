Mes Projets

Stage La Colinière

Lors de mon stage de 1ère année au lycée de la Colinière, mon projet était de finaliser une application de gestion des stages des étudiants, afin que le processus de gestion des stages soit simplifiés grâce à cette application. Ce logiciel était réalisé grâce à du PHP Objet.

L'application est composée de 3 interfaces utilisateurs :

- Une interface pour les Administrateurs

-Une interface pour les Professeurs

-Une interface pour les Elèves

L'interface pour les administrateurs avait pour objectif de pouvoir permettre toute la gestion des différents comptes de l'application, et d'avoir un contrôle total sur chaque utilisateur.

L'interface pour les professeurs leur permettait de pouvoir suivre les avancées de leurs élèves dans leur recherches de stage, et de pouvoir leur en proposer, ainsi que de pouvoir commentez et conseiller leurs différentes prospections.

L'interface pour les élèves leur permet de pouvoir consulter les offres de stages que leur propose leurs enseignants, de pouvoir prospecter, et de pouvoir confirmer leur stage une fois qu'ils en ont trouver un

Stage Université de Nantes

Le projet de mon stage de 2ème année était de réaliser un site dédié pour les vidéos que prenait les professeurs du laboratoire durant leur cours ou diverses représentations.

J'ai réaliser le site en entier à l'aide du CMS (Content Managment System) Drupal

Le site est composé d'une page dédiée aux vidéos à la manière de youtube, lorsque l'on clique sur une vidéos, elle s'affiche avec son descriptif, le nom du professeur qui l'a réalisée et sa catégorie

Le site contient également un calendrier où sont ajouter tous les événements à venir dans le laboratoire

FruitMix

Le projet FruitMix est en projet que j'ai réalisé durant ma première année de formation

Le but du projet était de remodelé un site déjà existant et d'accomplir les missions suivantes:

1. Installer la solution web, en local, sur son poste de travail avec WampServer

2. Tester le site déployer sur la station afin de consigner dans des fiches d’incident toutes les anomalies détectées et les suggestions de corrections à apporter

3. Proposer une nouvelle charte graphique à rédiger dans une documentation technique. Préciser entre autre les choix de : police, code des couleurs retenues, taille et formes des images, logo de l’entreprise, gestion des liens hypertextes,...

4. Modifier la feuille de style /css/principal.css en conséquence

5. Tester la qualité de présentation du site après modification, faire valider les choix par le chef de projet

6. Intégration du site sur un serveur Linux

7. Modifier la base de données afin d'y intégrer une table contact

8. Créer un formulaire de contact

9. Modifier la base de données afin d'ajouter une nouvelle page Cocktail sur le site
